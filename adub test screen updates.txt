Screen and backend rule mismatch notes

1. Admin is visible in public navigation.
- index.html exposes Admin Panel as a main screen.
- customer.html shows Admin in bottom navigation.
- provider.html shows Admin in bottom navigation.
- This violates the backend access rule because guest and subscriber flows should not be aware of admin access.
- Intended rule: admin access begins only after provider sign-in and admin sign/elevation from provider profile.

2. Provider requires sign-in but provider screen is exposed as an open dashboard.
- provider.html currently appears as a ready dashboard instead of a sign-in or gated provider entry.

3. Subscriber prompt is not aligned with expected business rule.
- customer.html currently presents guest request submission and optional priority payment language.
- Expected direction from user: subscriber prompt should appear for the monthly subscription fee.

4. Redesign deferred.
- These issues are being recorded now and left for redesign/alignment later.
- Current focus is getting the app behavior working before visual restructuring.
